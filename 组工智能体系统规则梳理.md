## 一、数据底座规则

### 1.1 数据来源与优先级

| 数据类型 | 权威来源 | 优先级 | 更新频率 | 责任科室 |
|---------|---------|--------|---------|---------|
| 干部基础信息 | 干部任免审批表 | ★★★★★ | 实时 |  |
| 任职经历 | 任免表 + 考察材料 | ★★★★★ | 实时 |  |
| 考核结果 | 年度考核登记表 | ★★★★☆ | 年度 |  |
| 民主测评 | 民主测评汇总表 | ★★★★☆ | 年度/任前 |  |
| 考察评价 | 考察材料/现实表现 | ★★★★☆ | 任前 |  |
| 述职报告 | 个人述职述德述廉 | ★★★☆☆ | 年度 |  |
| 奖惩记录 | 任免表奖惩栏 | ★★★★☆ | 实时 |  |
| 家庭成员 | 任免表家庭关系 | ★★★☆☆ | 实时 |  |

**冲突解决规则**：
- 同一字段多源冲突时，以"任免表"为准
- 任免表缺失时，以"考察材料"为准
- 时间字段冲突时，以组织部档案审核结果为准
- 数值冲突时（如票数），以原始测评表扫描件为准

### 1.2 数据质量校验规则

```
【必填项校验】
- 干部姓名：不能为空，必须为2-4个汉字
- 出生年月：格式YYYY-MM，年龄范围18-65
- 入党时间：不得晚于参加工作时间
- 参加工作时间：不得晚于当前日期
- 学历：必须从标准字典选择（博士/硕士/大学/大专/中专/高中）

【逻辑校验】
- 任职时间连续性：上一段结束时间 ≤ 下一段开始时间
- 职级晋升逻辑：科员→副科→正科→副处→正处，不得跳级
- 年龄与职级匹配：副处原则上≥35岁，正处原则上≥40岁
- 到龄预警：男60岁/女55岁到龄前6个月标红

【一致性校验】
- 任免表姓名 vs 考察材料姓名：必须一致
- 任免表职务 vs 分工备案职务：必须一致
- 民主测评人数 vs 应到人数：参会率≥80%为有效
```

#### 2.1.2 熟悉领域标签规则

**【专业干部标签判定规则】**

| 标签类型 | 判定条件（满足任一即可） | 数据源 |
|---------|----------------------|--------|
| **经济干部** | ①在发改委、财政局、国资委、投促中心、CBD管委会、金融办等经济部门任科长以上；②在街乡分管经济工作（经济发展办、经管站）；③有企业任职经历（国企班子成员） | 任免表 + 分工备案 |

**【标签冲突处理】**
- 一人可打多个标签（如"经济+党务"复合型干部）
- 标签优先级：经济 > 科技 > 规划 > 城建 > 党务 > 政法 > 教育 > 卫生
- 标签置信度：任免表直接证据=1.0，述职报告推断=0.8，考察材料旁证=0.7

#### 2.1.3 职务层次任职时间计算规则

```
【计算维度】
1. 科员级任职时长：参加工作至首次任副科时间
2. 副科级任职时长：首次任副科至首次任正科时间
3. 正科级任职时长：首次任正科至首次任副处时间
4. 副处级任职时长：首次任副处至首次任正处时间
5. 正处级任职时长：首次任正处至今

【晋升速度判定】（需与全库同层级均值比较）
- 快于均值20%以上 → "晋升较快"
- 均值±20%以内 → "晋升正常"
- 慢于均值20%以上 → "晋升较慢"

【岗位类别覆盖度】
- 党政：党委系统、政府系统
- 基层：街乡、社区（村）
- 机关：委办局
- 条块：垂直管理单位
- 企业：国企、事业单位
- 覆盖类型数量 ≥ 4 → "经历丰富"
- 覆盖类型数量 2-3 → "经历一般"
- 覆盖类型数量 ≤ 1 → "经历单一"
```

#### 2.2.2 标签提取规则

```
【提取来源权重】
1. 考察材料（上级撰写）：权重1.0，最权威，但可能放大执行类特点
2. 现实表现材料：权重0.95，客观性较强
3. 工作业绩事例：权重0.9，具体事例支撑
4. 三件典型事件：权重0.9，具体事例支撑
5. 述职报告（自述）：权重0.8，可能缺少客观性，需交叉验证
6. 民主测评结果：权重0.85，群众视角
7. 巡察报告：权重0.9，监督视角

【标签置信度计算】
- 直接描述（原文明确出现标签词）：confidence = 0.95-1.0
- 间接推断（需语义理解）：confidence = 0.80-0.94
- 单一来源：confidence × 0.9
- 多来源交叉验证：confidence × 1.0
- 自述标签（述职报告）：confidence × 0.85
- 他述标签（考察材料）：confidence × 1.0

【标签频次权重】
- 同一标签在3份以上材料中出现 → 权重 × 1.2
- 同一标签在2份材料中出现 → 权重 × 1.1
- 仅1份材料出现 → 权重 × 1.0

【矛盾标签处理】
- "果断"与"急躁"同时出现 → 标记为"多面性特征：果断但急躁"
- "严谨"与"灵活"同时出现 → 标记为"多面性特征：严谨且灵活"
- 正负面标签冲突 → 进入人工审核队列
```

#### 2.2.3 标签去重与归一规则

```
【同义词归一】
- "统筹协调能力强" = "善于统筹协调" = "统筹能力强" → 统一为"统筹协调能力强"
- "作风务实" = "作风扎实" = "求真务实" → 统一为"作风务实"
- "群众基础好" = "群众威信高" = "群众口碑好" → 统一为"群众基础好"

【上位词归并】（保留下位词）
- "善于处理复杂问题" → 归入"应急处突能力强"
- "善于化解矛盾" → 归入"群众工作能力强"
- "善于谋划" → 归入"思考谋划能力强"

【标签长度规范】
- 标准长度：4-8个汉字
- 过长截取：保留核心语义（如"在工作中表现出较强的统筹协调能力" → "统筹协调能力强"）
- 过短扩展：补充维度（如"果断" → "果断干练"）
```

### 2.3 工作实绩分析规则

#### 2.3.1 实绩提取标准

```
【事实要素三要素】（必须满足至少2项）
① 可量化指标：有具体数字（金额、人数、面积、百分比等）
② 第三方权威评价：上级批示、通报表扬、媒体报道、审计认定
③ 可验证成果物：牵头制定的文件、落地的具体项目、建立的机制

【实绩成色判定】
- 三要素全有 → "成效突出"（需业务人员确认）
- 满足2项 → "成效较好"
- 满足1项 → "成效一般"
- 无要素或仅有自述 → "成效待核实"

【实绩来源优先级】
1. 三件典型事件（最具体）
2. 工作业绩事例
3. 述职报告（可能有夸大）
4. 考察材料（可能较笼统）
```

#### 2.3.2 实绩分类规则

| 实绩类别 | 判定关键词 | 量化指标示例 | 权威评价示例 |
|---------|-----------|------------|------------|

| **民生改善** | 就业、教育、医疗、养老、住房 | 新增学位XX个、建设养老院XX家 | 群众满意度提升、12345投诉下降 |

### 3.1 班子结构分析规则

#### 3.1.1 年龄梯队规则

```
【年龄分段标准】
- 年轻干部：≤ 40岁（处级）/ ≤ 35岁（科级）
- 中年干部：41-50岁（处级）/ 36-45岁（科级）
- 年长干部：51-55岁（处级）/ 46-55岁（科级）
- 到龄干部：≥ 56岁（处级）/ ≥ 51岁（科级，女）/ ≥ 56岁（科级，男）

【理想结构比例】（街乡班子示例）
- 年轻干部：30-40%
- 中年干部：40-50%
- 年长干部：10-20%
- 到龄干部：≤ 10%（且应有接替人选）

【断层预警规则】
- 3年内到龄人数 ≥ 班子规模30% → "黄色预警：近期将出现集中退出现象"
- 5年内到龄人数 ≥ 班子规模50% → "橙色预警：中期班子将面临大规模调整"
- 无45岁以下干部 → "红色预警：班子严重老化，缺乏接续力量"
- 无50岁以下正职 → "红色预警：正职梯队断层"
```

#### 3.1.2 专业背景覆盖规则

```
【专业分类标准】
- 经济类：经济学、金融学、财政学、工商管理、统计学
- 法律类：法学、法律、政治学
- 城建类：城市规划、建筑学、土木工程、工程管理

- 卫生类：医学、公共卫生、护理学
- 其他类：上述未涵盖的专业

【理想覆盖标准】
- 经济类 ≥ 1人（负责经济发展）
- 法律类 ≥ 1人（负责依法行政）
- 城建类 ≥ 1人（负责城市建设）
- 党务类 ≥ 1人（负责党建工作）
- 专业覆盖率 = 实际覆盖类别 / 理想类别 ≥ 80%为"结构合理"
```

#### 3.1.3 经历结构规则

```
【经历类型分类】
- 机关经历：委办局工作经历
- 街乡经历：街道/乡/地区工作经历
- 基层经历：社区（村）、企事业单位一线工作经历
- 企业经历：国有企业、民营企业工作经历
- 挂职经历：上级机关、外地区挂职经历

【理想结构比例】（街乡班子）
- 有街乡经历：≥ 60%（熟悉基层）
- 有机关经历：≥ 40%（熟悉政策）
- 有基层经历：≥ 30%（接地气）
- 有企业经历：≥ 10%（懂经济）
- 有挂职经历：≥ 10%（视野宽）

#### 3.1.4 任职年限分布规则

```
【任职年限分段】
- 新任（< 1年）：需要适应期，不宜大幅调整
- 成熟（1-3年）：工作进入状态，可承担重任
- 稳定（3-5年）：经验丰富，是班子骨干
- 长期（> 5年）：可能产生倦怠，建议轮岗

【稳定性指数计算】
- 近3年班子成员变动次数（调入+调出+晋升+免职）
- 变动率 = 变动人次 / 班子规模
- 变动率 < 0.3 → "班子稳定"
- 变动率 0.3-0.5 → "班子正常调整"
- 变动率 > 0.5 → "班子变动频繁，需关注"
```

### 3.3 配备方案模拟推演规则

#### 3.3.1 模拟推演维度

```
【调整前后对比指标】
1. 平均年龄变化：↓ 表示年轻化，↑ 表示老龄化
2. 专业覆盖度：新增专业类别数
3. 经历互补性：新增经历类型数
4. 性别比例变化：女性比例变化
5. 学历结构变化：研究生比例变化
6. 稳定性预测：调整后3年内到龄人数

【连锁影响分析】
- 调整A岗位 → 可能影响B岗位的接替链条
- 提拔A → 可能空缺A的原岗位，需同步考虑
- 调入A → 需考虑A与原班子成员的工作关系、专业重叠

【风险识别规则】
- 回避关系：新调入人员与现有班子成员有夫妻关系、直系血亲关系、三代以内旁系血亲关系 → 自动标红
- 专业重叠：新调入人员与现有班子成员专业背景完全相同 → 黄色提示
- 风格冲突：新调入人员与正职风格标签冲突（如"强势主导型"+"强势主导型"） → 橙色提示
- 年龄倒挂：副职年龄 > 正职年龄5岁以上 → 黄色提示
```

#### 3.3.2 方案评分规则

```
【综合评分公式】
综合得分 = 年龄结构得分 × 25% + 专业覆盖得分 × 25% + 
           经历互补得分 × 20% + 稳定性得分 × 15% + 
           风险规避得分 × 15%

【各维度评分标准】
- 年龄结构得分：越接近标准模型得分越高（满分100）
- 专业覆盖得分：覆盖类别越多得分越高（满分100）
- 经历互补得分：经历类型越丰富得分越高（满分100）
- 稳定性得分：3年内到龄人数越少得分越高（满分100）
- 风险规避得分：无回避关系、无风格冲突得满分

【方案排序规则】
- 综合得分 > 85 → "推荐方案"
- 综合得分 70-85 → "可行方案"
- 综合得分 < 70 → "需谨慎考虑"
```

### 4.1 岗位画像构建规则

#### 4.1.1 岗位类型标注规则

| 岗位类型 | 界定标准 | 典型岗位 | 核心能力要求 |
|---------|---------|---------|------------|
| **攻坚型** | 承担急难险重任务、历史遗留问题多、矛盾集中 | 信访办主任、拆迁办主任、重点街乡书记 | 应急处突、群众工作、担当精神 |
| **守成型** | 工作基础好、需要保持稳定、风险防控为主 | 成熟街乡书记、稳定委办局正职 | 统筹协调、规矩意识、稳健作风 |
| **转型型** | 面临改革任务、需要创新发展、突破瓶颈 | 新组建单位负责人、功能区管委会 | 改革创新、思考谋划、学习能力 |
| **综合协调型** | 涉及面广、需要多方协调、服务领导决策 | 区委办、区政府办、发改委 | 统筹协调、沟通配合、政治素质 |
| **专业业务型** | 专业性强、需要深耕领域、技术支撑 | 财政局、规自分局、科信局 | 专业能力、严谨细致、依法办事 |

#### 4.1.2 岗位核心能力要求推导规则

```
【历任干部特征统计法】
1. 提取该岗位近5任干部的画像标签
2. 统计高频标签（出现频率 > 60%）
3. 将这些高频标签作为岗位核心能力要求

【三定方案推导法】
1. 提取三定方案中的职责描述关键词
2. 将关键词映射到能力标签体系
   - "统筹协调各部门工作" → "统筹协调能力强"
   - "负责经济发展工作" → "经济工作经历"
   - "处理群众来信来访" → "群众工作能力强"
   - "推进重点项目建设" → "执行力强、敢于担当"

【区委重点工作调整法】
- 当前区委重点工作为"数字经济" → 该岗位增加"数字化能力"要求
- 当前区委重点工作为"城市更新" → 该岗位增加"城建工作经历"要求
- 权重调整：与重点工作相关的能力要求权重 × 1.5
```

#### 4.1.3 岗位任职资格规则

```
【刚性条件】（一票否决）
- 政治面貌：必须是中共党员（除明确规定外）
- 学历：必须符合岗位最低要求（如正处原则上本科以上）
- 任职年限：必须符合《干部任用条例》规定年限
  * 提任副处：正科满3年
  * 提任正处：副处满2年
  * 提任副局：正处满3年
- 年龄：不得超过任职年龄上限
- 廉洁：无正在受到党纪政务处分
- 考核：近3年考核无"基本称职"及以下

【柔性条件】（综合评分）
- 专业匹配：专业背景与岗位要求匹配度
- 经历匹配：工作经历与岗位要求匹配度
- 能力匹配：能力标签与岗位要求匹配度
- 性格匹配：性格特征与岗位类型匹配度
```

### 4.2 以岗找人匹配规则

#### 4.2.1 匹配引擎规则

```
【双引擎匹配流程】
Step 1: 规则引擎过滤（一票否决）
  - 筛选符合刚性条件的干部
  - 不符合任一刚性条件 → 直接排除
  - 输出：候选人池

Step 2: 语义匹配引擎（综合评分）
  - 将岗位要求转化为标签向量
  - 将候选人画像转化为标签向量
  - 计算余弦相似度
  - 输出：匹配度排名

【匹配度计算公式】
匹配度 = 经历匹配度 × 30% + 能力匹配度 × 35% + 
         性格匹配度 × 20% + 成长匹配度 × 15%

【经历匹配度计算】
- 岗位所需经历类型 ∩ 候选人经历类型
- 重合度 = 重合类型数 / 岗位所需类型数
- 关键岗位经历匹配：每匹配1个 + 10分
- 专业领域经历匹配：每匹配1个 + 8分

【能力匹配度计算】
- 岗位核心能力要求 ∩ 候选人能力标签
- 重合度 = 重合标签数 / 岗位核心标签数
- 高频标签匹配：每匹配1个 + 10分
- 低频标签匹配：每匹配1个 + 5分

【性格匹配度计算】
- 攻坚型岗位 → 优先"急先锋"、"敢于担当"、"果断干练"
- 守成型岗位 → 优先"老黄牛"、"稳健"、"细致"
- 转型型岗位 → 优先"改革创新"、"学习能力强"、"视野开阔"
- 综合协调型 → 优先"统筹协调"、"沟通能力强"、"灵活"
- 专业业务型 → 优先"严谨细致"、"专业能力强"、"依法办事"

【成长匹配度计算】
- 候选人当前能力段位 vs 岗位历任干部平均段位
- 高于均值 → "发挥台"（能充分发挥能力）
- 接近均值 → "锻炼台"（能提升能力）
- 低于均值 → "风险岗"（可能难以胜任）
```

#### 4.2.2 推荐理由生成规则

```
【推荐理由模板】
"推荐{姓名}担任{岗位}，理由如下：
① 经历匹配：该同志具有{XX}经历，与岗位要求高度契合。
   具体事例：{从业绩事例/三件典型事件中提取}
② 能力匹配：该同志{能力标签}突出，符合岗位核心要求。
   具体表现：{从考察材料中提取}
③ 性格适配：该同志{性格标签}，适合{岗位类型}岗位特点。
④ 成长空间：该同志当前处于{发挥台/锻炼台}，到任后预计{预期效果}。
⑤ 风险提醒：{如有短板则提示，如"需注意加强经济工作学习"}"

【过滤规则】
- 不得出现干部个人隐私信息（如家庭住址、联系方式）
- 不得出现未经核实的负面评价
- 不得出现与组织工作口径不符的表述
- 不得出现歧视性语言
```

### 4.3 以人找岗匹配规则

#### 4.3.1 反向匹配逻辑

```
【能力缺口分析】
- 提取候选人能力标签
- 提取岗位要求能力标签
- 计算缺口 = 岗位要求标签 - 候选人已有标签
- 输出：能力缺口报告

【岗位推荐排序】
1. 匹配度最高的3个岗位
2. 每个岗位附带：
   - 匹配度得分
   - 能力缺口分析
   - 推荐理由
   - 能力提升建议

【职业发展路径建议】
- 当前岗位 → 建议下一步岗位 → 远期目标岗位
- 路径设计原则：
  * 补齐短板：先到能补充能力缺口的岗位
  * 发挥优势：再到能充分发挥优势的岗位
  * 循序渐进：每步提升1个层级或1个维度
```

#### 4.3.2 能力缺口报告模板

干部：李某

③ 专业能力：无城建专业背景

最优路径：
当前（分管民生）→ 某街道副主任（分管经济，补齐短板）→ 
某委办局副职（发挥综合协调能力）→ 正职（全面锻炼）
```

## 五、知识问答式搜索规则

### 5.1 自然语言解析规则

```
【查询条件识别】
- 年龄条件："35岁以下" → age ≤ 35
- 职级条件："正科级" → position_level = "正科"
- 经历条件："有拆迁经验" → 标签包含"拆迁"或经历包含"拆迁办"

- 地域条件："有街乡经历" → 经历包含"街道/乡/地区"

【语义歧义处理】

- "年轻干部" → 系统追问："是指35岁以下还是40岁以下？"

【条件组合逻辑】
- "且"关系：默认，所有条件必须同时满足
- "或"关系：用户明确说"或"时启用
- "排除"关系：用户说"不包括"、"排除"时启用

1. '{负面标签}'
   核实方向：请了解改进情况和现实表现
   建议问题：{生成具体问题}

四、需补充的实绩
1. 画像中'{实绩类别}'较为薄弱
   核实方向：请了解近三年的具体工作成果
   建议问题：{生成具体问题}"
```

### 6.2 谈话记录整理规则

```
【信息提取】
1. 从谈话记录中提取新的标签
2. 从谈话记录中提取新的实绩
3. 从谈话记录中核实已有标签
4. 从谈话记录中修正错误标签

【与画像比对】
- 谈话证实标签 → 提升置信度
- 谈话否定标签 → 标记"需删除"，进入审核
- 谈话新增标签 → 标记"待确认"
- 谈话修正标签 → 标记"待确认"

【完整性评估】
- 谈话是否覆盖了所有疑点？
- 谈话是否核实了所有低置信度标签？
- 谈话是否了解了干部的主要不足和改进情况？
- 未覆盖项 → 生成补充谈话建议
```

## 七、反馈闭环机制规则

### 7.1 反馈类型与处理

| 反馈类型 | 业务人员操作 | 系统处理 | 影响 |
|---------|-----------|---------|------|
| **有用** | 点击"有用" | 记录标签使用次数+1 | 提升标签权重 |
| **无用** | 点击"无用" | 记录标签忽略次数+1 | 降低标签权重，低于阈值自动隐藏 |
| **错误** | 点击"错误"并修正 | 记录错误类型，进入纠错队列 | 修正标签，更新语料库 |
| **补充** | 手动添加新标签 | 记录新增标签，进入审核 | 补充画像 |

### 7.2 模型效果评估规则

```
【月度评估指标】
1. 标签准确率 = 正确标签数 / 总标签数 × 100%（目标 ≥ 85%）
2. 标签召回率 = 提取标签数 / 应提取标签数 × 100%（目标 ≥ 80%）
3. 匹配推荐命中率 = 被采纳推荐数 / 总推荐数 × 100%（目标 ≥ 60%）
4. 用户满意度 = 有用反馈数 / 总反馈数 × 100%（目标 ≥ 80%）

【优化触发条件】
- 准确率连续2月 < 80% → 触发模型微调
- 某类标签准确率 < 70% → 补充该类语料
- 匹配命中率连续3月 < 50% → 调整匹配算法权重
```

## 八、安全与合规规则

### 8.1 数据分级分类

| 数据级别 | 内容 | 处理策略 |
|---------|------|---------|
| **绝密** | 干部个人隐私（身份证号、住址、联系方式） | 脱敏处理，系统不存储，仅档案室可查 |
| **机密** | 干部考核原始票、考察谈话原始记录 | 加密存储，仅限授权人员查看 |
| **秘密** | 干部画像标签、匹配推荐结果 | 内网存储，按权限访问 |
| **内部** | 班子结构分析、岗位画像 | 内网存储，业务人员可查看 |

### 8.2 访问权限规则

| 角色 | 权限范围 |
|-----|---------|
| 系统管理员 | 系统配置、用户管理、日志审计 |
| 部领导 | 查看所有画像、班子分析、匹配推荐 |
| 干部一科/二科 | 维护分管范围干部画像、查看班子分析 |
| 考核科 | 维护考核数据、查看民主测评分析 |
| 监督科 | 查看负面信息、处理处分记录 |
| 街乡组织委员 | 查看本单位班子分析、维护本单位干部信息 |

### 8.3 输出内容审核规则

```
【自动过滤内容】
- 干部个人隐私信息（身份证号、家庭住址、电话号码）
- 未经核实的负面评价（无来源支撑的）
- 涉及干部家庭矛盾的敏感信息
- 与组织工作口径不符的表述
- 歧视性、侮辱性语言

【人工审核触发】
- 涉及干部提拔、调整的建议
- 涉及班子配备的重大调整方案
- 涉及干部处理的负面信息
- 系统置信度 < 0.85 的推荐
```
