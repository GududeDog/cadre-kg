# 规则清单与注入逻辑说明

> 用途：定义所有可执行的抽取规则和标签注入规则，作为 seeds/*.cypher 的编写依据
> 配合文档：标签体系定义_V1.md、标签-字段映射表.md、训练数据_全量文档_结构化字段提取分析v1.md
> 时间：2026-05-29

---

## 一、规则体系总览

```
规则引擎
├── 第一层：文档类型识别规则（15条）
│   └── 路径关键词 + 内容指纹 + 结构特征 → doc_type
│
├── 第二层：字段抽取规则（按ExtractionProfile，每个doc_type一组）
│   └── 表格单元格 → field:value 映射（任免表/分工表/考察登记表/民主测评）
│
├── 第三层：标签注入规则（~55条）
│   ├── 3A. 字段值规则（25条）
│   ├── 3B. 履历规则（10条）
│   ├── 3C. 关键词/语义规则（15条）
│   └── 3D. 统计计算规则（5条）
│
└── 第四层：展示优先级与去重规则（4条）
```

### 2.2 分类流程

```
文件路径+文件名
    ↓ 路径关键词匹配（权重0.4）
命中≥2个 → 置信度+0.8
    ↓ 标题/首段内容关键词匹配（权重0.3）
命中≥3个 → 置信度累计
    ↓ 结构特征验证（权重0.3）
表格数/段落数/表头匹配 → 置信度累计
    ↓
置信度≥0.85 → 直接分类
0.60≤置信度<0.85 → LLM分类（传DocumentFingerprint，不传全文）
置信度<0.60 → 标记unknown
```

## 十、规则版本管理

保存在图数据库的 `(:Rule)` 节点中：

```cypher
(:Rule {
  rule_id: "KW-01",
  rule_name: "组织协调能力关键词规则",
  rule_type: "semantic",
  target_entity: "Cadre",
  target_tag: "组织协调能力强",
  dimension: "能力特点",
  condition_json: '{"keywords":["统筹","协调","兼顾","全局"],"doc_types":["annual_report","assessment_material"]}',
  priority: 5,
  confidence_default: 0.75,
  enabled: true,
  version: "v1",
  created_at: "2026-05-29"
})

CREATE (r)-[:APPLIES_TO_DOC_TYPE]->(:DocType {doc_type:"annual_report"});
CREATE (r)-[:APPLIES_TO_DOC_TYPE]->(:DocType {doc_type:"assessment_material"});
CREATE (r)-[:PRODUCES_TAG]->(:Tag {tag_name:"组织协调能力强"});
```
